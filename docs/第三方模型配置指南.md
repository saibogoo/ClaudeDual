## 一、这是什么？为什么要用？

Claude Desktop 于 2026 年 4 月悄悄上线了 **Developer Mode**（开发者模式）。Anthropic 没有发布博客、没有搞发布会，只是更新了技术文档。

开启后，你可以将 Claude Desktop 的底层推理模型替换为任何兼容 Anthropic Messages API 的第三方模型——包括 Kimi K2.6、DeepSeek、GPT-5、Gemini，甚至是本地部署的开源模型。

### 核心价值

- **配额自由**：不再受 Claude 官方每日配额限制，第三方模型按 API 用量计费
- **成本降低**：以 Kimi K2.6 Coding 套餐为例，API 成本约为 Claude 的 1/5
- **框架保留**：Claude Desktop 的 Code 模式、Cowork 模式、MCP 连接、上下文压缩、沙箱隔离等框架级能力全部保留
- **灵活切换**：通过隔离实例，原版和第三方模式可以并行使用

> 💡 简单来说：Claude Desktop 真正值钱的不是某一个模型，是这套 agent 基础设施。现在你可以保留这套框架，换上任何你想用的"脑子"。

## 三、推荐方式：使用 ClaudeDual

**ClaudeDual** 是专为此场景设计的 macOS 管理工具，将开发者模式开启、第三方推理配置、隔离实例启动三步合并为一个 App，无需手动修改任何配置文件。

### 3.1 安装 ClaudeDual

1. 前往 [ClaudeDual GitHub Releases](https://github.com/saibogoo/ClaudeDual/releases) 下载最新 DMG
2. 打开 DMG，将 `ClaudeDual.app` 拖入 Applications 文件夹
3. 首次运行如提示"无法验证开发者"，在终端执行：

```bash
sudo xattr -r -d com.apple.quarantine /Applications/ClaudeDual.app
```

### 3.2 配置流程

**第一步：开启开发者模式**

打开 ClaudeDual，在「状态」标签页点击 **"一键开启开发者模式"**。ClaudeDual 会自动创建隔离数据目录并写入所需配置，无需手动操作 Claude Desktop 菜单。

**第二步：填写推理配置**

切换到「配置」标签页，新建或编辑配置：

| 字段 | Kimi K2.6 示例值 |
|------|-----------------|
| 配置名称 | Kimi K2.6 |
| 代理模式 | 本地代理 |
| API Base URL | `https://api.moonshot.ai/v1` |
| API Key | 你的 Moonshot API Key |
| 上游认证方式 | Bearer |
| 上游模型名称 | `kimi-k2` |

填写完成后点击 **"创建并激活"**（新建）或 **"保存配置"**（编辑）。

> 💡 如果你使用 [CC-Switch](https://github.com/musistudio/ccswitch) 作为本地网关，可以将代理模式切换为 **CC Switch**，直接填写 CC-Switch 的地址（默认 `http://127.0.0.1:15721`），模型映射和认证统一在 CC-Switch 中管理。

**第三步：启动隔离实例**

回到「状态」标签页，点击 **"启动"** 按钮。ClaudeDual 会：

1. 启动内置代理服务器（本地代理模式）
2. 将推理配置写入隔离数据目录
3. 以 `--user-data-dir` 参数启动独立的 Claude Desktop 实例

启动后，新实例与你原本的 Claude Desktop 完全隔离，两者可以同时运行。

## 四、手动方式（进阶参考）

如果你不想使用 ClaudeDual，也可以手动完成整个流程。

### 4.1 开启 Developer Mode

1. 打开 Claude Desktop
2. 点击菜单栏 **Help → Troubleshooting**
3. 点击 **Enable Developer Mode**
4. 菜单栏出现新的 **Developer** 菜单，确认开启成功

### 4.2 配置第三方推理

*以 Kimi K2.6 为例*

1. 点击菜单栏 **Developer → Configure Third-Party Inference**
2. **Connection** 区域选择 **Gateway（Anthropic-compatible）**
3. 填写：
   - **API Base URL**：`https://api.moonshot.ai/v1`
   - **API Key**：你的 Moonshot API Key
4. **Models** 区域填入：`kimi-k2`
5. 点击 **Apply locally** 保存
6. 完全退出 Claude Desktop 并重新打开
7. 启动界面选择 **"Continue with Gateway"** 或 **"Start in Cowork on 3P"**

> 💡 使用手动方式时，第三方配置会影响你的主 Claude Desktop 实例。若需保留原版和第三方并行，需要按下方步骤手动配置隔离实例。

### 4.3 手动隔离双开

如需在不影响主实例的前提下运行第三方模式，可在终端执行：

```bash
open -n -a /Applications/Claude.app --args \
  --user-data-dir="$HOME/Library/Application Support/ClaudeDual-3p"
```

首次启动后，需在该隔离实例中手动重新配置 Developer Mode 和第三方推理（步骤同 4.1、4.2）。隔离实例的配置存储在独立目录中，与主实例互不影响。

## 五、第三方模式功能对照表

切换到第三方模式后，Claude Desktop 的框架级能力大部分保留，但依赖 Anthropic 服务器的功能不可用。

### ✅ 可用功能

| 功能 | 说明 |
|------|------|
| Code 模式 | 完整的编程环境：文件读写、代码 diff、Bash 终端 |
| Cowork 模式 | 桌面 agent：任务规划、子代理调度、文件生成 |
| 本地 MCP 服务器 | stdio 方式的本地 MCP 正常工作 |
| 远程 MCP | 通过 MDM 或手动配置的远程 MCP 服务器 |
| 文件工具链 | Read / Edit / Write / Bash / Grep / Glob |
| WebSearch / WebFetch | 内置的网页搜索和抓取工具 |
| 多会话管理 | Desktop UI 层面的多任务并行 |
| 上下文自动压缩 | 长对话不断片，关键状态自动保留 |
| 沙箱隔离 | Code 模式系统级隔离，Cowork 虚拟机隔离 |
| CLAUDE.md | 项目级配置文件，压缩时优先保留 |
| 子代理调度 | 派生子代理并行处理子任务 |
| Git Worktree | 每个会话独立的 Git 工作树 |

### ❌ 不可用功能

| 功能 | 原因 |
|------|------|
| Chat 模式 | 第三方模式下 Chat Tab 被禁用 |
| Computer Use | 依赖 Anthropic 服务器的屏幕控制能力 |
| 内置 Connectors | Google Drive / Slack 等一键连接器依赖 Anthropic 基础设施 |
| Voice Mode | 语音对话功能依赖 Anthropic 服务 |
| Claude in Chrome | 浏览器扩展依赖 Anthropic 后端 |
| Dispatch（手机派任务） | 需要 Anthropic 账户和云端服务 |
| Skills / Plugin 市场 | 在线市场不可用，但支持本地分发的插件 |

## 六、Cowork 模式简介

Cowork 是 Claude Desktop 内置的桌面 agent 模式，面向非开发者用户设计。受限于 Claude 账号在国内使用的不稳定性，很多人可能还没怎么用过这个功能，但它其实非常强——与普通的 AI 聊天助手有本质区别。

### 与其他 AI 工具的核心差异

| 对比维度 | ChatGPT / Copilot | Claude Cowork |
|---------|-------------------|---------------|
| 交互模式 | 你问一句，它答一句 | 你下达目标，它自主规划执行 |
| 输出形式 | 聊天窗口里的文字 | 真实的文件（Word/PPT/Excel/PDF） |
| 文件访问 | ChatGPT Agent 跑在云端虚拟机，碰不到本地文件 | 直接操作本地文件，但在沙箱里隔离 |
| 任务复杂度 | 单轮对话 | 多步骤拆解 + 子代理并行 |
| 安全机制 | 各平台不同 | 独立虚拟机沙箱，只能访问授权文件夹 |

### Cowork 典型使用场景

- **竞品分析报告**：指定文件夹 + 告诉它研究对象，自动生成完整 Word 报告
- **数据整理**：把散落的 CSV / Excel 文件汇总分析，输出可视化报告
- **文档批处理**：批量读取合同/简历，提取关键信息到 Excel 表格
- **PPT 制作**：给出提纲和素材文件夹，自动生成演示文稿

## 七、常见问题 FAQ

**Q: 开启 Developer Mode 后找不到 "Configure Third-Party Inference"？**

请确认 Claude Desktop 版本 ≥ v1.2581.0。更新后需要完全退出并重新打开应用。部分 Team 计划用户反馈该选项可能被管理员限制。

**Q: 第三方模式下能用 Anthropic 的模型吗？**

可以。你可以通过 Amazon Bedrock、Google Vertex AI 或 Azure AI Foundry 接入 Claude 模型，推理走你自己的云服务商，Anthropic 不会看到对话内容。

**Q: Windows 可以用隔离双开吗？**

ClaudeDual 仅支持 macOS。Windows 用户可以通过创建不同的快捷方式并添加 `--user-data-dir` 参数实现类似效果。

**Q: 第三方模式下数据安全吗？**

第三方模式下，所有推理请求直接发送到你配置的 API 端点，不经过 Anthropic 服务器。对话数据存储在本地设备上。Anthropic 仅收集使用量遥测（token 计数等），且可以完全禁用。

**Q: 能用本地部署的模型吗？**

可以。只要你的本地推理服务器实现了 Anthropic Messages API 格式（`/v1/messages` 端点），就可以通过 Gateway 模式接入。推荐使用 vLLM 或 LiteLLM 作为代理。

**Q: 第三方模式是正式功能还是测试功能？**

目前标注为 Research Preview（研究预览），功能可能会变化。但 Amazon Bedrock、Google Vertex AI、Microsoft Foundry 已官方支持该部署方式。

**Q: ClaudeDual 会修改我的主 Claude Desktop 配置吗？**

不会。ClaudeDual 使用独立的数据目录（`~/Library/Application Support/ClaudeDual-3p`），与主实例完全隔离。停止 ClaudeDual 管理的实例后，主实例不受任何影响。
